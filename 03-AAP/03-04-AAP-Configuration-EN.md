## 2. Deploy Starter Pack V3

> **Purpose**: Prepare trusted execution for AIOps — bulk-create demo automation objects in AAP via IaC.

### 2.1 Overview

Starter Pack V3 uses **Infrastructure as Code (IaC)** for air-gapped deployment:

1. Stage offline ZIPs and collection tarballs on the AAP Controller host
2. Run `AAP_Starter_Pack_v3.sh` as `admin`
3. Script extracts content, installs collections offline, runs playbooks in fixed order

**IaC provisions:**

| Type | Description |
| --- | --- |
| **Organization** | `StarterPack` |
| **Projects** | Starter Pack scenario projects |
| **Inventories / Hosts / Groups** | Managed hosts and groups |
| **Credentials** | Connection credentials |
| **Job Templates** | Templates with surveys (expandable to 32+) |

You may also run the same playbooks manually in the same order for debugging.

### 2.2 Architecture & Directory Layout

**Target environment:**

| Item | DEMO config |
| --- | --- |
| **AAP version** | 2.6 Container Growth (single node) |
| **Execution context** | AAP Controller host, Linux user `admin` |
| **Projects root** | `/home/admin/aap/controller/data/projects` |

**Post-deployment layout:**

| Path | Purpose |
| --- | --- |
| `Starter_Pack_V3/` | Extracted scenarios, READMEs; collections copied into selected scenario dirs |
| `Iac_For_StaterPack_V3/` | IaC playbooks, `configs/*.yml`, local `./collections`, `ansible.cfg` |

**Key IaC files:**

| File / dir | Description |
| --- | --- |
| `ansible.cfg` | Generated by script (`collections_paths = ./collections`, etc.) |
| `configure_aap.yml` | Organization + Projects (dispatch entry) |
| `configure_projects.yml` | Projects |
| `configure_inventories.yml` | Inventories → Hosts → Groups |
| `configure_credentials.yml` | Credentials |
| `configure_job_templates.yml` | Job Templates (with surveys) |
| `configs/` | `auth.yml`, `organizations.yml`, `projects.yml`, `inventories.yml`, `hosts.yml`, `groups.yml`, `credentials.yml`, `job_templates.yml` |

**Generated `ansible.cfg` excerpt:**

```ini
[defaults]
inventory = ./inventory
remote_user = root
collections_paths = ./collections
deprecation_warnings = False
host_key_checking = False

[privilege_escalation]
become = false
become_method = sudo
become_user = root
become_ask_pass = false
```

### 2.3 Prerequisites

#### Offline ZIP files

Place these 4 files in `Offline_installation_packages/` **alongside** `AAP_Starter_Pack_v3.sh` (script resolves `${SCRIPT_DIR}/Offline_installation_packages`):

| File | Description |
| --- | --- |
| `01_AnsibleCollectionForStarterPack_IaC.zip` | IaC project + collection tarballs for offline `ansible-galaxy` install |
| `02_APAC_AAP_Starter_Pack-main.zip` | Starter Pack scenario tree |
| `03_collections.zip` | Extra collections for scenario use |
| `04_IaC_offline_packages.zip` | Additional IaC offline payload (may include sample configs) |

#### Host directory prep

```bash
# root: create offline package dir and upload ZIPs
mkdir -p /tmp/Offline_installation_packages/
chmod 755 /tmp/Offline_installation_packages/*.zip

# upload script and make executable
chmod 755 /tmp/AAP_Starter_Pack_v3.sh

# confirm Projects dir exists (script precheck fails if missing)
ls /home/admin/aap/controller/data/projects
```

#### AAP API variables

Playbooks need Controller API settings. The one-click script passes `-e` extra vars; for manual runs maintain `configs/auth.yml`:

```yaml
---
aap_hostname: aap26.example.com
aap_username: admin
aap_password: "redhat"
aap_validate_certs: false   # use true + valid certs in production

# Production: OAuth token instead of password
# aap_token: "your-oauth-token"
```

> Use OAuth token + TLS validation in production; never commit real secrets to public repos.

#### Offline collections (example versions)

Script installs into `./collections` (versions per bundle):

- `ansible-controller`, `ansible-hub`, `ansible-eda`, `ansible-platform`
- `ansible-posix`, `kubernetes-core`
- `infra-aap_configuration`, `infra-aap_configuration_extended`, `infra-aap_utilities`, `infra-ah_configuration`

Verify after install:

```bash
cd /home/admin/aap/controller/data/projects/Iac_For_StaterPack_V3
ansible-galaxy collection list
```

### 2.4 IaC Design Principles

| Principle | Description |
| --- | --- |
| **Separation of data and logic** | AAP objects in `configs/*.yml`; playbooks load and call `infra.aap_configuration` roles |
| **Composable execution** | `configure_aap.yml` for Org + Projects; run object-specific playbooks for incremental debugging |

### 2.5 Config Files & Role Mapping

| Config file | Top-level key | Role |
| --- | --- | --- |
| `organizations.yml` | `aap_organizations` | `gateway_organizations` |
| `projects.yml` | `controller_projects` | `controller_projects` |
| `inventories.yml` / `hosts.yml` / `groups.yml` | — | `controller_inventories` / `controller_hosts` / `controller_host_groups` |
| `credentials.yml` | `controller_credentials` | — (encrypt secrets with Ansible Vault) |
| `job_templates.yml` | `controller_templates` | `controller_job_templates` |

> Survey defaults must be strings where AAP expects them (e.g. multi-choice `"true"` / `"false"`).

### 2.6 One-Click Deployment (recommended)

#### Step 1: Edit script variables

```bash
su - admin
cd /tmp/
vi AAP_Starter_Pack_v3.sh
```

DEMO example at top of script:

```bash
ADMIN_USER="admin"                    # must match user running the script
AAP_HOSTNAME="aap26.example.com"      # AAP Controller FQDN
AAP_USER="admin"                      # AAP API user
AAP_PASSWORD="redhat"                 # use strong secret or Vault in production
```

#### Step 2: Run script

```bash
./AAP_Starter_Pack_v3.sh
```

**Script phases:**

| Phase | Action |
| --- | --- |
| **Precheck** | Current user = `ADMIN_USER`; 4 ZIPs present; `PROJECTS_DIR` exists |
| **Starter Pack** | Create `Starter_Pack_V3`, extract `02_*.zip` / `03_*.zip`, copy collections into scenario dirs |
| **IaC** | Create `Iac_For_StaterPack_V3`, extract `01_*.zip` / `04_*.zip`; backup/restore `configs/auth.yml` if present |
| **Collections** | `ansible-galaxy collection install … -p ./collections` |
| **Playbooks** | Run in strict order below |

**Playbook order:**

```bash
ansible-playbook configure_aap.yml           -e "aap_hostname=... aap_username=... aap_password=..."
ansible-playbook configure_credentials.yml   -e "aap_hostname=... aap_username=... aap_password=..."
ansible-playbook configure_projects.yml      -e "aap_hostname=... aap_username=... aap_password=..."
ansible-playbook configure_inventories.yml   -e "aap_hostname=... aap_username=... aap_password=..."
ansible-playbook configure_job_templates.yml -e "aap_hostname=... aap_username=... aap_password=..."
```

**Success message:**

```
==== AAP Starter Pack v3 offline one-click deployment completed.
     Please log in to AAP Web UI to verify that objects were created as expected. ====
```

> Full runs produce lengthy unzip and Ansible output; archive logs if audit evidence is needed.
